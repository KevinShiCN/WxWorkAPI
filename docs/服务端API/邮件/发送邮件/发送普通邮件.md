## 发送普通邮件

发送邮件接口，支持带附件
**请求方式：** POST（**HTTPS**）
**请求地址：** https://qyapi.weixin.qq.com/cgi-bin/exmail/app/compose_send?access_token=ACCESS_TOKEN

**请求包体**

      
```
{
    "to": {
        "emails": [
            "word@bgjkcsqy.wecom.work",
            "jyan1@tiyantest.wang",     
        ],
        "userids": [
            "william"
        ]
    },
    "cc": {
        "emails": [],
        "userids": ["panyy"]
    },
    "bcc": {
        "emails": ["zoro@bgjkcsqy.wecom.work"],
        "userids": []
    }
    "subject": "这是标题",
    "content": "这是邮件正文",
    "attachment_list": [
        {
            "file_name": "a.txt",
            "content": "BASE64_CONTENT"
        }
    ]
}
```

    

**参数说明**

        

| 参数 | 必须 | 类型 | 说明 |
| --- | --- | --- | --- |
| to | 是 | obj | 收件人 定义见收件人字段，to.emails 和 to.userids 至少得传一个 |
| cc | 否 | obj | 抄送 定义见收件人字段 |
| bcc | 否 | obj | 密送 定义见收件人字段 |
| subject | 是 | string | 标题 |
| content | 是 | string | 内容 |
| attachment_list | 否 | obj[] | 附件相关 |
| content_type | 否 | string | 内容类型 html，text（默认是html） |

** 收件人字段 **

        

| 参数 | 必须 | 类型 | 说明 |
| --- | --- | --- | --- |
| emails | 否 | string[] | 邮箱地址 |
| userids | 否 | string[] | userid |

**附件二级扩展字段 attachment_list **

        

| 参数 | 必须 | 类型 | 说明 |
| --- | --- | --- | --- |
| file_name | 是 | string | 文件名 |
| content | 是 | string | 文件内容（base64加密），所有附件加正文的大小不允许超过50M, 且附件个数不能超过200个 |

 

**权限说明**

- 自建应用需配置到“可调用应用”列表中的应用secret所获取的accesstoken来调用（accesstoken如何获取？）
- 第三方应用需具有“邮件”权限
- 代开发自建应用需具有“邮件”权限

**返回示例**

      
```
{
    "errcode": 0,
    "errmsg": "ok"
}
```

    

**参数说明**

        

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |

 

## 发送日程邮件

**请求方式：** POST（**HTTPS**）
**请求地址：** https://qyapi.weixin.qq.com/cgi-bin/exmail/app/compose_send?access_token=ACCESS_TOKEN

**请求包体**

      
```
{
    "to": {
        "emails": [
            "word@bgjkcsqy.wecom.work",
            "jyan1@tiyantest.wang",
            "zorocheng@tencnet.com",
            "zoro@bgjkcsqy.wecom.work"
        ],
        "userids": [
            "william"
        ]
    },
    "cc": {
        "emails": [],
        "userids": ["panyy"]
    },
    "bcc": {
        "emails": [],
        "userids": []
    }
    "subject": "这是邮件标题",
    "content": "这是邮件正文，也是日程描述",
    "schedule": {
        "method": "request",
        "location": "这是地点",
        "subject": "日程标题",
        "is_all_day_event": 0,
        "start_time": 1669278600,
        "end_time": 1669282200,
        "timezone": 8,
        "reminders": {
            "is_remind": 1,
            "remind_before_event_mins": 15,
            "is_repeat": 1,
            "repeat_interval": 1,
            "repeat_type": 1,
            "repeat_day_of_week": [
                4,
                5
            ],
            "repeat_day_of_month": [

            ],
            "repeat_week_of_month": [

            ],
            "repeat_month_of_year": [

            ],
            "repeat_until": 1670745600
        }
    }
}
```

    

**参数说明**

        

| 参数 | 必须 | 类型 | 说明 |
| --- | --- | --- | --- |
| to | 是 | obj | 收件人 定义见收件人字段，to.emails 和 to.userids 至少得传一个 |
| cc | 否 | obj | 抄送 定义见收件人字段 |
| bcc | 否 | obj | 密送 定义见收件人字段 |
| subject | 是 | string | 标题 |
| content | 是 | string | 内容 |
| schedule | 否 | obj | 日程相关；有这个字段表示这是日程 |
| meeting | 否 | obj | 会议相关；有这个字段表示这是会议，此时必须同时带上schedule，会议的基本设置放在schedule里 |
| attachment_list | 否 | obj[] | 附件相关 |
| content_type | 否 | string | 内容类型 html，text（默认是html） |

** 收件人字段 **

        

| 参数 | 必须 | 类型 | 说明 |
| --- | --- | --- | --- |
| emails | 否 | string[] | 邮箱数组 |
| userids | 否 | string[] | userid数组 |

**日程二级扩展字段 schedule**

        

| 参数 | 必须 | 类型 | 说明 |
| --- | --- | --- | --- |
| schedule_id | 否 | string | 日程ID (修改/取消日程必须带上schedule_id) |
| method | 否 | string | 日程方法： request-请求（不传schedule_id时是创建日程，传了是修改日程） cancel-取消日程（必须带上schedule_id） 默认为request |
| subject | 是 | string | 日程标题 |
| location | 是 | string | 地点 |
| is_all_day_event | 否 | int | 是否全天事件 0-否 1-是 |
| start_time | 是 | int | 日程开始时间，Unix时间戳 |
| end_time | 是 | int | 日程结束时间，Unix时间戳 |
| timezone | 否 | int | 时区。UTC偏移量表示(即偏离零时区的小时数)，东区为正数，西区为负数。 例如：+8 表示北京时间东八区默认为北京时间东八区 取值范围：-12 ~ +12 |
| reminders | 否 | obj | 重复和提醒相关字段 |
| schedule_admins | 否 | obj | 日程管理员字段, 定义见收件人字段, 允许传多个用户，上限是3个，只支持传userid，必须是同企业的用户，且在参与人中 |

**日程重复和提醒字段 reminders**

        

| 参数 | 必须 | 类型 | 说明 |
| --- | --- | --- | --- |
| is_remind | 否 | int | 是否有提醒 0-不提醒 1-提醒 |
| remind_before_event_mins | 否 | int | 日程开始（start_time）前多少分钟提醒，当is_remind=1时有效。例如：15表示日程开始前15分钟提醒-15表示日程开始后15分钟提醒 |
| is_repeat | 否 | int | 是否重复 0-否 1-是 |
| repeat_type | 否 | int | 重复类型，当is_repeat=1时有效。目前支持如下类型：0 - 每日1 - 每周2 - 每月5 - 每年 |
| repeat_interval | 否 | int | 重复间隔该字段随repeat_type不同而含义不同例如：repeat_interval指定为2，repeat_type指定为每周重复，那么每2周重复一次；repeat_interval指定为2，repeat_type指定为每月重复，那么每2月重复一次 |
| repeat_day_of_week | 否 | int[] | 每周周几重复仅当重复类型为每周时有效取值范围：1 ~ 7，分别表示周一至周日 |
| repeat_week_of_month | 否 | int[] | 每月哪周重复仅当重复类型为每月时有效取值范围：1 ~ 4，分别表示第1-4周另外 -2 表示最后一周 |
| repeat_day_of_month | 否 | int[] | 每月哪几天重复仅当重复类型为每月时有效取值范围：1 ~ 31，分别表示1~31号 |
| repeat_month_of_year | 否 | int[] | 每年哪几个月重复仅当重复类型为每年时有效取值范围：1 ~ 12，分别表示 1月 - 12月 |
| repeat_until | 否 | int | 重复结束时刻，Unix时间戳，当is_repeat=1时有效。不填或填0表示一直重复 |

**权限说明**

- 自建应用需配置到“可调用应用”列表中的应用secret所获取的accesstoken来调用（accesstoken如何获取？）
- 第三方应用需具有“邮件”权限
- 代开发自建应用需具有“邮件”权限

**返回示例**

      
```
{
    "errcode": 0,
    "errmsg": "ok"
}
```

    

**参数说明**

        

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| errcode | int32 | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |

| enter_restraint | 否 | int | 是否限制成员入会  0:所有人可入会 1:仅登录用户可入会 2:仅企业内部用户可入会；默认所有人可入会 |
