# 获取会议发起记录

该接口用于获取员工发起的会议记录及其状态。

**请求方式：** POST（**HTTPS**）
**请求地址：** https://qyapi.weixin.qq.com/cgi-bin/meeting/initiate/list?access_token=ACCESS_TOKEN

**请求包体：**

      
```
{
   "type": 1,
   "begin_time": 1700000000,
   "end_time": 1700001000,
   "limit": 1000,
   "cursor": "CURSOR"
}
```

    

**参数说明：**

        

| 参数 | 必须 | 类型 | 说明 |
| --- | --- | --- | --- |
| access_token | 是 | string | 调用接口凭证。获取方法查看“[获取access_token] |
| type | 是 | int | 查询类型。1:发起成功的会议记录2:发起失败的会议记录 |
| begin_time | 是 | int | 查询范围起始时间戳，单位为秒 |
| end_time | 是 | int | 查询范围结束时间戳，单位为秒 |
| limit | 否 | int | 每次拉取的数据量，默认值200，最大值1000 |
| cursor | 否 | string | 用于分页查询的游标，由上一次调用返回，首次调用可不填 |

**补充说明:**
会过滤会议创建者不在应用可见范围中的收款记录，因此返回的记录数可能会小于limit设置的最大记录数。

**权限说明：**

- 会议创建者需要在应用可见范围内
- 自建应用需要配置在“可调用接口的应用”列表

**返回结果：**

      
```
{
	"errcode": 0,
	"errmsg": "ok",
	"next_cursor": "abcd",
	"meeting_list": [
		"meetingid": "MEETINGID",
		"userid": "USERID",
		"initiate_time": 1700000000
	]
}
```

    

**参数说明：**

        

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| errcode | int | 返回码 |
| errmsg | string | 对返回码的文本描述内容 |
| next_cursor | string | 当前数据最后一个key值，如果下次调用带上该值则从该key值往后拉，用于实现分页拉取 |
| has_more | bool | 是否还有数据待拉取 |
| meeting_list | object[] | 发起会议成功或失败的记录信息，取决于请求中的type参数 |
| meeting_list.meetingid | string | 会议ID |
| meeting_list.userid | string | 会议发起者的userid |
| meeting_list.initiate_time | int | 会议发起时间 |
