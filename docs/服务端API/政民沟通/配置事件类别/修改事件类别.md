# 修改事件类别

可以修改已配置的一级或二级事件类别，网格员在提交「巡查上报」或「居民上报」时将填入配置的事件类别

**请求方式：** POST（**HTTPS**）
**请求地址：** https://qyapi.weixin.qq.com/cgi-bin/report/grid/update_cata?access_token=ACCESS_TOKEN

**请求包体：**

```
{
    "category_id": "category_id",
    "category_name": "category_name",
    "level" : 2,
    "parent_category_id": "parent_category_id"
}

```

**参数说明：**

[
| 参数 | 必须 | 说明 |
| --- | --- | --- |
| access_token | 是 | 调用接口凭证。“获取access_token” |
| category_id | 是 | 分类id |
| category_name | 是 | 分类名称，不能超过30个字,同一一级分类下的二级分类名字不能一样 |
| level | 是 | 分类层级，这里只能传1或者2 |
| parent_category_id | 否 | 所属的一级分类的id，level为2的话必传 |

**权限说明：**

- “居民联系” secret默认可使用网格接口
- 自建应用需要配置在“居民联系” 的“可调用接口的应用”中

**返回结果：**

```
{
   "errcode": 0,
   "errmsg": "ok",

}

```

**参数说明：**

| 参数 | 说明 |
| --- | --- |
| errcode | 返回码 |
| errmsg | 对返回码的文本描述内容 |
